▶서브쿼리 사용 시 서브쿼리 내용을 외부커리와 반대로 바꿨을 때 에러가 뜨는 이유와 설명

SELECT 고객이름, 적립금 FROM 고객 WHERE 적립금 = (SELECT MAX(적립금) FROM 고객);
>정상 표기

SELECT MAX(적립금), 적립금 FROM 고객 WHERE 적립금 = (SELECT 고객이름 FROM 고객);
>에러

서브쿼리에서 SELECT 고객이름 FROM 고객를 사용하여 고객의 이름을 반환하고, 외부 쿼리에서 해당 이름과 일치하는 적립금을 찾으려는 것으로 이해했습니다. 
그러나 두 번째 쿼리에 문제가 있습니다.

두 번째 쿼리에서는 서브쿼리가 SELECT MAX(적립금), 적립금 FROM 고객 WHERE 적립금 = (SELECT 고객이름 FROM 고객)로 구성되어 있습니다. 
여기서 문제가 되는 부분은 SELECT 고객이름 FROM 고객의 서브쿼리 결과가 단일 값인 고객의 이름을 반환하는 것이 아니라는 점입니다. 
서브쿼리에서는 단일 값이 아닌 다중 값이 반환되어야 합니다.

따라서 두 번째 쿼리를 수정해야 합니다. 적립금이 가장 많은 고객의 이름을 찾아야 하므로, 서브쿼리에서는 고객 테이블에서 최대 적립금을 찾는 것이 맞습니다. 
수정된 쿼리는 다음과 같습니다.

SELECT 고객이름, 적립금
FROM 고객
WHERE 적립금 = (SELECT MAX(적립금) FROM 고객);
